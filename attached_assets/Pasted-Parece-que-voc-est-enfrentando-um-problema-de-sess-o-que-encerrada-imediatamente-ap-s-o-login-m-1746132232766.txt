Parece que você está enfrentando um problema de sessão que é encerrada imediatamente após o login, mesmo que a autenticação seja bem-sucedida. Isso pode acontecer por vários motivos. Vamos analisar algumas possíveis causas e soluções:

### Possíveis Causas e Soluções:

1. **Configuração de Sessão Incorreta**  
   - Verifique as configurações de sessão no seu sistema (ex.: tempo de vida da sessão, armazenamento, cookies).  
   - Se estiver usando PHP, por exemplo, verifique `session.gc_maxlifetime` no `php.ini`.  

2. **Problema com Cookies**  
   - Se o sistema usa cookies para manter a sessão, verifique se:  
     - O navegador está bloqueando cookies.  
     - O domínio/caminho do cookie está configurado corretamente.  
     - O cookie está sendo sobrescrito ou excluído após o login.  

3. **Redirecionamento Pós-Login**  
   - Pode haver um redirecionamento incorreto após o login, fazendo com que a sessão seja perdida.  
   - Verifique se há algum código que limpa a sessão após o redirecionamento.  

4. **Problema no Armazenamento da Sessão**  
   - Se as sessões estão armazenadas em um banco de dados ou sistema de arquivos, verifique permissões e se os dados estão sendo gravados corretamente.  

5. **Conflito em Cache/Sessão**  
   - Limpe o cache do navegador e tente novamente.  
   - Teste em modo anônimo/privado para descartar conflitos de extensões.  

6. **Problema no Código de Autenticação**  
   - Verifique se, após a autenticação, a sessão está sendo corretamente inicializada e mantida.  
   - Exemplo em PHP:  
     ```php
     session_start(); // Deve ser chamado no início do script
     $_SESSION['user_id'] = $user_id; // Exemplo de armazenamento
     ```

7. **Problema com HTTPS/SSL**  
   - Se o site usa HTTPS, verifique se o cookie de sessão está configurado como `Secure` e `SameSite` corretamente.  

### Como Diagnosticar Melhor:
- **Verifique os logs do servidor** para erros relacionados à sessão.  
- **Inspecione os cookies** (F12 > Application > Cookies) e veja se eles persistem após o login.  
- **Teste em outro navegador/dispositivo** para ver se o problema persiste.  

Se você compartilhar mais detalhes (linguagem, framework, mensagens de erro), posso ajudar com uma solução mais específica!
